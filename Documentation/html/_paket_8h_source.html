217/*
218general packet-structure:
219 ================================================================
220|| recipient | referenceid | type | size | payload ||
221 ================================================================
222
223 recipient: INTEGER | 0 = broadcast
224 referenceid: INTEGER
225 type: BYTE(CHAR)
226 size: INTEGER [OPTIONAL]
227 payload: CHAR* (bytearray) [OPTIONAL]
228
229 It's possible to stack a packet into another one.
230
231 ==============================================================================================
232
233 FORWARD-packet:
234
235 <recipient> <referenceid> <forward-byte-code> <size of forwarded packet> <packet to forward>
236
237 --> forwarded packet can also be another forward packet, so that way it is possible
238 to send the packet over a predefined route.
239
240 SIZE: 3*INTEGER + 1 + forward_packet_size
241
242 ==============================================================================================
243
244 ACK-packet:
245
246 <recipient> <referenceid> <ACK-byte-code>
247 --> through the reference id the recipient of this packet is assured, that his message
248 arrived at its destination
249
250 SIZE: 2*INTEGER + 1
251
252 ==============================================================================================
253
254 WHOHAS-packet:
255
256 <recipient = 0> <referenceid> <WHOHAS-byte-code> <maximum hops [INTEGER]> <address of the wanted client [INTEGER]>
257
258 --> repeated by everyone who receives this. the referenceid assures that a packet isn't
259 repeated twice by the same client.
260
261 maximum hops: maximum count of hops to destination
262 SIZE: 4*INTEGER + 1
263
264 ==============================================================================================
265
266 IHAVE-packet:
267
268 <recipient> <referenceid> <IHAVE-byte-code> <maximum hops> <ihave_grid_size> <ihavegrid>
269
270 --> this packet is the answer to the WHOHAS packet and contains an ihavegrid.
271
272 ihavegrid:
273 contains a possible network route (consisting of hops) to the desired client.
274 a hop is defined by its address and reachability(signal strength).
275
276 maximum hops: maximum count of hops in this possible route, if a node gets an ihave packet
277 it only add itself to the ihavegrid and forwards the packet if the ihave grid
278 doesnt already have the maximum count of hops and also doesn't contain this
279 node yet.. that garantees that ihave packets
280 dont circle in the aether forever since they are broadcasted and reproduced
281
282 if several IHAVE-packets are received the best route can be determined using the hop-data
283 of every single hop.
284
285 SIZE: 3*INTEGER + 1 + gridsize
286
287
288 ==============================================================================================
289
290 MSG-packet:
291
292 <recipient> <referenceid> <MSG-byte-code> <routesize> <used route> <payloadsize> <payload>
293
294 routesize: INTEGER | number of bytes, not hops
295 used route: INTEGER[]
296 --> the route should also contain the sender adress so that the ack can be directed
297
298 SIZE: 4*INTEGER + 1 + routesize + payloadsize
299
300 ==============================================================================================
301
302 typebytes:
303 0 - whohas
304 1 - ihave
305 2 - msg
306 3 - ACK
307 4 - forward
308*/
